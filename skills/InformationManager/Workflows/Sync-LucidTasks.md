# Sync-LucidTasks Workflow

Sync LucidTasks tasks into Kaya's context.

**Category:** Sync (Source → Kaya)
**Trigger:** `sync lucidtasks`, `refresh tasks`, `update task context`, `/info tasks`

## Purpose

Pull current LucidTasks data FROM the SQLite DB INTO `~/.claude/context/LucidTasksContext.md`.

This is a **Sync** workflow: it reads from the local LucidTasks SQLite database and writes
Kaya's context directory.

## Domain Configuration

| Setting | Value |
|---------|-------|
| **DB Path** | `~/.claude/skills/LucidTasks/Data/lucidtasks.db` |
| **API** | `kaya-cli tasks --json` |
| **Output** | `~/.claude/context/LucidTasksContext.md` |

## Execution Steps

### 1. Fetch Tasks via CLI

```bash
kaya-cli tasks --json 2>/dev/null
```

### 2. Parse Response

Extract:
- Task names and due dates
- Project assignments
- Completion status
- Priority indicators

### 3. Write Context File

Write to `~/.claude/context/LucidTasksContext.md`:

```markdown
---
tags: [context, ai-context, lucidtasks]
last_updated: {{TIMESTAMP}}
gathered_by: InformationManager
entries_count: {{TASK_COUNT}}
---

# LucidTasks Context

## Summary
{{TASK_COUNT}} active tasks across {{PROJECT_COUNT}} projects.

## Tasks by Project

### {{Project Name}}
- [ ] Task 1 (due: YYYY-MM-DD)
- [x] Task 2 (completed)
...

## Overdue Tasks
{{List any overdue tasks}}

## Upcoming (Next 7 Days)
{{List tasks due in next week}}

---
*Generated by InformationManager/Workflows/Sync-LucidTasks.md*
```

### 4. Return Summary

Output:
- Number of tasks fetched
- Number of projects
- Any errors encountered
