# UpdateArchitecture Workflow

**Purpose:** Regenerate all architecture diagrams from current system state. Use this after making changes to skills, hooks, or memory structure.

---

## Workflow Steps

### Step 1: Gather Fresh System State

```bash
# Count current skills
SKILL_COUNT=$(ls -d ~/.claude/skills/*/ 2>/dev/null | wc -l | tr -d ' ')

# Count current hooks
HOOK_COUNT=$(ls ~/.claude/hooks/*.ts 2>/dev/null | wc -l | tr -d ' ')

# List skill names
SKILLS=$(ls -d ~/.claude/skills/*/ | xargs -n1 basename | sort)

# List hook names
HOOKS=$(ls ~/.claude/hooks/*.ts | xargs -n1 basename | sed 's/\.ts$//' | sort)

# Check memory directories
MEMORY_DIRS=$(ls -d ~/.claude/MEMORY/*/ 2>/dev/null | xargs -n1 basename)

echo "Skills: $SKILL_COUNT"
echo "Hooks: $HOOK_COUNT"
echo "---"
echo "$SKILLS"
echo "---"
echo "$HOOKS"
```

### Step 2: Detect Changes Since Last Generation

```bash
# Check if previous architecture exists
if [ -f ~/.claude/USER/PAI_ARCHITECTURE.md ]; then
  LAST_GEN=$(head -5 ~/.claude/USER/PAI_ARCHITECTURE.md | grep "Generated:" | cut -d: -f2-)
  echo "Last generated: $LAST_GEN"
fi

# Find recently modified skills
find ~/.claude/skills -name "SKILL.md" -mtime -7 -exec dirname {} \; | xargs -n1 basename

# Find recently modified hooks
find ~/.claude/hooks -name "*.ts" -mtime -7 -exec basename {} \;
```

### Step 3: Regenerate All Documents

Execute each workflow in sequence:

1. **GenerateArchitecture** → `~/.claude/USER/PAI_ARCHITECTURE.md`
2. **SessionLifecycle** → `Output/SESSION_LIFECYCLE.md`
3. **SkillMap** → `Output/SKILL_MAP.md`
4. **MemoryStructure** → `Output/MEMORY_STRUCTURE.md`

### Step 4: Create Update Summary

Add to the top of each document:

```markdown
**Last Updated:** [TIMESTAMP]
**Previous Update:** [from file if exists]
**Changes Detected:**
- [List of new/modified skills]
- [List of new/modified hooks]
- [New memory directories]
```

### Step 5: Create Index File

Create/update `~/.claude/skills/Content/SystemFlowchart/Output/INDEX.md`:

```markdown
# Kaya Architecture Documentation Index

**Last Full Update:** [TIMESTAMP]

## Generated Documents

| Document | Description | Last Updated |
|----------|-------------|--------------|
| [Kaya_ARCHITECTURE.md](Kaya_ARCHITECTURE.md) | Master system overview | [timestamp] |
| [SESSION_LIFECYCLE.md](SESSION_LIFECYCLE.md) | Hook execution flow | [timestamp] |
| [SKILL_MAP.md](SKILL_MAP.md) | Skill hierarchy | [timestamp] |
| [MEMORY_STRUCTURE.md](MEMORY_STRUCTURE.md) | Memory system | [timestamp] |

## Quick Stats

- **Skills:** [count]
- **Hooks:** [count]
- **Memory Directories:** [count]

## Recent Changes

[List of skills/hooks modified since last update]

---

*Generated by SystemFlowchart skill*
```

### Step 6: Announce Completion

```bash
curl -s -X POST http://localhost:8888/notify \
  -H "Content-Type: application/json" \
  -d '{"message": "All architecture diagrams updated successfully"}' \
  > /dev/null 2>&1 &
```

Open index:
```bash
open ~/.claude/skills/Content/SystemFlowchart/Output/INDEX.md
```

---

## Validation Checklist

After regeneration, verify:

- [ ] All skill counts match actual `skills/` directory
- [ ] All hook counts match actual `hooks/` directory
- [ ] New skills appear in SKILL_MAP.md
- [ ] New hooks appear in SESSION_LIFECYCLE.md
- [ ] Memory directories match actual MEMORY/ structure
- [ ] All Mermaid diagrams render correctly
- [ ] Timestamps are updated in all documents
- [ ] INDEX.md reflects current state

---

## When to Run This Workflow

**Run update when:**
- New skill created
- Hook added or modified
- Memory structure changed
- Configuration updated
- After major Kaya upgrade
- Weekly maintenance (recommended)

**Trigger phrases:**
- "update architecture diagrams"
- "refresh system flowchart"
- "regenerate Kaya documentation"
