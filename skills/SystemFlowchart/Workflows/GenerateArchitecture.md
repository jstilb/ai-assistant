# GenerateArchitecture Workflow

**Purpose:** Generate comprehensive Kaya system architecture documentation with dual output (Mermaid markdown + PNG images).

---

## Overview

This workflow generates both:
1. **Mermaid markdown** - Quick rendering in GitHub/Obsidian/VS Code, version-controllable
2. **PNG images** - Professional visuals via Art skill's TechnicalDiagrams workflow

Output structure:
```
Output/
├── markdown/           # Mermaid .md files
│   ├── system-overview.md
│   ├── session-lifecycle.md
│   ├── skill-ecosystem.md
│   ├── memory-structure.md
│   ├── configuration-flow.md
│   └── agent-architecture.md
├── images/             # Art-generated PNGs
│   └── *.png
└── .cache-state.json   # Staleness tracking
```

---

## Workflow Steps

### Step 1: Check Cache Status

First, check if regeneration is needed:

```bash
bun ~/.claude/skills/SystemFlowchart/Tools/CacheManager.ts check
```

- **Exit 0** = Cache is stale, regeneration needed → continue
- **Exit 1** = Cache is fresh, skip regeneration (unless forced)

To force regeneration:
```bash
bun ~/.claude/skills/SystemFlowchart/Tools/CacheManager.ts invalidate
```

### Step 2: Scan System State

Use SystemScanner to gather current system data:

```bash
# Full system scan (for reference)
bun ~/.claude/skills/SystemFlowchart/Tools/SystemScanner.ts scan

# Or specific components
bun ~/.claude/skills/SystemFlowchart/Tools/SystemScanner.ts skills
bun ~/.claude/skills/SystemFlowchart/Tools/SystemScanner.ts hooks
bun ~/.claude/skills/SystemFlowchart/Tools/SystemScanner.ts memory
```

### Step 3: Categorize Skills

Use SkillCategorizer to organize skills into Meta/Orchestration/Specialized:

```bash
bun ~/.claude/skills/SystemFlowchart/Tools/SkillCategorizer.ts
```

Output shows skills grouped by category with categorization reasoning.

### Step 4: Generate Mermaid Diagrams

Use DiagramBuilder to generate all markdown diagrams:

```bash
# Generate all diagrams and save to Output/markdown/
bun ~/.claude/skills/SystemFlowchart/Tools/DiagramBuilder.ts all

# Or generate individual diagrams
bun ~/.claude/skills/SystemFlowchart/Tools/DiagramBuilder.ts overview
bun ~/.claude/skills/SystemFlowchart/Tools/DiagramBuilder.ts lifecycle
bun ~/.claude/skills/SystemFlowchart/Tools/DiagramBuilder.ts ecosystem
bun ~/.claude/skills/SystemFlowchart/Tools/DiagramBuilder.ts memory
bun ~/.claude/skills/SystemFlowchart/Tools/DiagramBuilder.ts config
bun ~/.claude/skills/SystemFlowchart/Tools/DiagramBuilder.ts agents
```

### Step 5: Generate PNG Images (Optional)

For professional visual diagrams, use ArtBridge to generate PNGs:

```bash
# Generate all diagrams as PNGs
bun ~/.claude/skills/SystemFlowchart/Tools/ArtBridge.ts all --output ~/.claude/skills/SystemFlowchart/Output/images

# Or convert markdown diagrams to PNGs
bun ~/.claude/skills/SystemFlowchart/Tools/ArtBridge.ts batch \
  --input ~/.claude/skills/SystemFlowchart/Output/markdown \
  --output ~/.claude/skills/SystemFlowchart/Output/images
```

### Step 6: Update Master Architecture Document

Create the master document at `~/.claude/skills/CORE/USER/PAI_ARCHITECTURE.md`:

```markdown
# Kaya System Architecture

**Generated:** [TIMESTAMP]
**System Hash:** [hash from CacheManager]

## Quick Stats

| Metric | Count |
|--------|-------|
| Skills | [from scan] |
| Hooks | [from scan] |
| Workflows | [from scan] |
| Tools | [from scan] |

| Category | Purpose | Count |
|----------|---------|-------|

## Diagrams

[embed system-overview.md or link to PNG]

### Session Lifecycle
[embed session-lifecycle.md or link to PNG]

[embed memory-structure.md or link to PNG]

[embed configuration-flow.md or link to PNG]

[embed agent-architecture.md or link to PNG]

```

### Step 7: Update Cache

After successful generation:

```bash
bun ~/.claude/skills/SystemFlowchart/Tools/CacheManager.ts update
```

### Step 8: Voice Notification

Announce completion:

```typescript
import { notifySync } from '~/.claude/skills/CORE/Tools/NotificationService.ts';
notifySync("Kaya architecture diagrams generated successfully");
```

## Skill Categories

### Meta Skills
Skills about skills/system - infrastructure, configuration, visualization:
- CORE, System, SystemFlowchart, SkillAudit, CreateSkill
- PAISync, KayaUpgrade, GeminiSync

### Orchestration Skills
Coordination/execution engines - multi-agent, workflows, scheduling:
- THEALGORITHM, Agents, Council, RedTeam
- AutoMaintenance, KnowledgeMaintenance, ProactiveEngine
- QueueRouter, AutonomousWork, _RALPHLOOP

### Specialized Skills
Domain-specific functionality - everything else:
- Research, Obsidian, Browser, Art, Shopping, etc.

- [ ] Master document at `CORE/USER/PAI_ARCHITECTURE.md` is current

## CLI Quick Reference

```bash
# Full workflow
bun CacheManager.ts check && bun DiagramBuilder.ts all && bun CacheManager.ts update

# Force regeneration
bun CacheManager.ts invalidate && bun DiagramBuilder.ts all && bun CacheManager.ts update

# With PNG generation
bun DiagramBuilder.ts all && bun ArtBridge.ts all && bun CacheManager.ts update

# Check status only
bun CacheManager.ts status
bun SystemScanner.ts scan
bun SkillCategorizer.ts
```

## Integration

### Uses
- `Tools/SystemScanner.ts` - Scan skills, hooks, memory
- `Tools/SkillCategorizer.ts` - Categorize into Meta/Orchestration/Specialized
- `Tools/DiagramBuilder.ts` - Generate Mermaid diagrams
- `Tools/ArtBridge.ts` - Generate PNG via Art skill
- `Tools/CacheManager.ts` - Track staleness

### Feeds Into
- `CORE/USER/PAI_ARCHITECTURE.md` - Master architecture document
- System skill integrity checks
- Kaya onboarding documentation
