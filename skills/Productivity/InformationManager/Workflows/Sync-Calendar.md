# Sync-Calendar Workflow

Sync calendar events into Kaya's context.

**Category:** Sync (Source → Kaya)
**Trigger:** `sync calendar`, `refresh calendar`, `update calendar context`, `/info calendar`

## Purpose

Pull upcoming calendar events FROM Google Calendar INTO `~/.claude/context/CalendarContext.md`.

This is a **Sync** workflow: it pulls data from Google Calendar into Kaya's context directory.

## Domain Configuration

| Setting | Value |
|---------|-------|
| **API** | `kaya-cli gcal` |
| **Output** | `~/.claude/context/CalendarContext.md` |

## Execution Steps

### 1. Fetch Events via CLI

```bash
kaya-cli gcal agenda --days 7 2>/dev/null
```

### 2. Parse Response

Extract:
- Event titles and times
- Calendar sources
- Attendees (if available)
- Recurring event indicators

### 3. Write Context File

Write to `~/.claude/context/CalendarContext.md`:

```markdown
---
tags: [context, ai-context, calendar]
last_updated: {{TIMESTAMP}}
gathered_by: InformationManager
entries_count: {{EVENT_COUNT}}
---

# Calendar Context

## Summary
{{EVENT_COUNT}} events in the next 7 days.

## Today ({{DATE}})
- 09:00 - Event 1
- 14:00 - Event 2

## Tomorrow ({{DATE}})
- 10:00 - Event 3

## This Week
### {{DAY}}
- Events...

---
*Generated by InformationManager/Workflows/Sync-Calendar.md*
```

### 4. Return Summary

Output:
- Number of events fetched
- Events today
- Any errors encountered
