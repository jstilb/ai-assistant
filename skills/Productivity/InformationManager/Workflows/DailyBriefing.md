# Daily Briefing Workflow

Generate and deliver a comprehensive morning briefing with multi-channel output.

**Trigger:** `morning briefing`, `daily briefing`, `what's on today`, `start my day`

## Purpose

Aggregate context from multiple sources and deliver a personalized morning briefing to help Jm start the day with full situational awareness.

## Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                    ProactiveEngine                               │
│                    (8am daily cron)                              │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    InformationManager                            │
│                    DailyBriefingGenerator.ts                     │
```

## Data Sources

| Source | File/Command | Context Captured |
|--------|--------------|------------------|
| Calendar | `context/CalendarContext.md` | Today's events, free windows |
| Tasks | `context/LucidTasksContext.md` *(generated by Sync-LucidTasks workflow)* | Due today, this week, upcoming |
| Goals | `USER/TELOS/GOALS.md`, `USER/TELOS/MISSIONS.md` | Active WIGs, missions |
| Patterns | `MEMORY/LEARNING/SIGNALS/ratings.jsonl` | Rating trends, top patterns |

## Delivery Channels

| Channel | Tool | Output |
|---------|------|--------|
| **Voice** | `lib/core/NotificationService.notifySync()` | Spoken 30-second summary |
| **Written** | File write | `MEMORY/BRIEFINGS/YYYY-MM-DD.md` |
| **Push** | ntfy | Quick glance notification |
| **Telegram** | `skills/Communication/Telegram/Tools/TelegramClient.ts send` | Full mobile reference |

## Execution Steps

### 1. Trigger (Automated or Manual)

**Automated:** ProactiveEngine cron job at 8am daily
```yaml
# MEMORY/daemon/cron/jobs/daily-briefing.yaml
id: daily-briefing
schedule: "0 8 * * *"
type: isolated
task: |
  bun ~/.claude/skills/Productivity/InformationManager/Tools/DailyBriefingGenerator.ts
output: silent
enabled: true
```

**Manual:** User request
```
User: "Give me my morning briefing"
→ Run DailyBriefingGenerator.ts
```

### 2. Gather Context

The generator reads from existing context files (refreshed by other workflows):

```bash
# Ensure context is fresh (optional pre-step)
bun ~/.claude/skills/Productivity/InformationManager/Tools/AgentGatherer.ts --sources calendar,asana,telos,learnings
```

### 3. Generate Briefing

```bash
# Full generation with all deliveries
bun ~/.claude/skills/Productivity/InformationManager/Tools/DailyBriefingGenerator.ts

# Preview without sending
bun ~/.claude/skills/Productivity/InformationManager/Tools/DailyBriefingGenerator.ts --dry-run

# JSON output for programmatic use
bun ~/.claude/skills/Productivity/InformationManager/Tools/DailyBriefingGenerator.ts --json
```

### 4. Deliver to Channels

All channels receive simultaneously:

1. **Voice** - Spoken summary (30 seconds max)
2. **Written** - Full markdown saved to `MEMORY/BRIEFINGS/`
3. **Push** - One-line notification via ntfy
4. **Telegram** - Formatted message for mobile reference

## Prerequisites

1. **Context files should be reasonably fresh**
   - Run `/info calendar` if CalendarContext.md is stale
   - Run `/info lucidtasks` if LucidTasksContext.md is stale

2. **Notification services configured**
   - Voice server running on localhost:8888
   - ntfy topic configured in settings.json
   - Telegram bot configured in secrets.json

3. **TELOS structure exists**
   - `USER/TELOS/GOALS.md`
   - `USER/TELOS/MISSIONS.md`

## Troubleshooting

**"No calendar events loaded"**
→ Run `/info calendar` to refresh CalendarContext.md

**"Voice delivery failed"**
→ Check voice server: `curl http://localhost:8888/health`

**"Telegram delivery failed"**
→ Verify secrets.json has telegram.bot_token and telegram.chat_id

**"No tasks loaded"**
→ Run `/info lucidtasks` to refresh LucidTasksContext.md
