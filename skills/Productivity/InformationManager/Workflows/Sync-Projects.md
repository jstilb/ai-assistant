# Sync-Projects Workflow

Sync projects context from the projects directory into Kaya's context.

**Category:** Sync (Source → Kaya)
**Trigger:** `sync projects`, `refresh projects`, `update projects context`, `/info projects`

## Purpose

Scan project directories and write summary TO `~/.claude/context/ProjectsContext.md`.

This is a **Sync** workflow: it pulls project data into Kaya's context directory.

## Domain Configuration

| Setting | Value |
|---------|-------|
| **Projects Location** | `/Users/[user]/Desktop/projects/` |
| **Output** | `~/.claude/context/ProjectsContext.md` |

## Execution Steps

### 1. Scan Directory

```bash
ls -d ~/Desktop/projects/*/ 2>/dev/null
```

For each project, check for:
- README.md
- package.json (Node.js/TypeScript)
- Cargo.toml (Rust)
- go.mod (Go)
- .git directory

### 2. Extract Project Info

For each project directory:
- Name
- Tech stack (based on config files)
- Has README
- Last commit date (if git repo)
- Brief description (from README first line or package.json)

### 3. Write Context File

Write to `~/.claude/context/ProjectsContext.md`:

```markdown
---
tags: [context, ai-context, projects]
last_updated: {{TIMESTAMP}}
gathered_by: InformationManager
entries_count: {{PROJECT_COUNT}}
---

# Projects Context

## Summary
{{PROJECT_COUNT}} projects in workspace.

## Projects

| Project | Tech Stack | README | Last Activity |
|---------|------------|--------|---------------|
| lucidview | TypeScript | Yes | 2026-02-03 |
| pai | TypeScript | Yes | 2026-02-02 |
...

## By Tech Stack

### TypeScript/Node.js
- lucidview
- pai
- ...

### Rust
- ...

### Go
- ...

## Recent Activity

Projects with commits in last 7 days:
- lucidview (3 commits)
- pai (5 commits)

---
*Generated by InformationManager/Workflows/Sync-Projects.md*
```

### 4. Return Summary

Output:
- Total projects found
- Projects by tech stack
- Recent activity count
- Any errors encountered
